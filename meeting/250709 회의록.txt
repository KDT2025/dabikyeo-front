2. 개발
- 기획적인 부분을 보여주는 것이 우선 > 개발
- GPS 위도, 경도 값 비교 
  >> 구급차 반경 1km 이내의 모든 유저들에게 알림을 전송
- 2~5초 간격으로 위치 검색 및 갱신
- 현재 사용자가 몇 차선에 있는지
 아이폰 Carplay 
- 구급차에서 어플을 켠 상 태에서 말을 하면 그 내용이 일반 사용자의 앱을 통해 전달됨
  어느 차선 비워달라고 하면 알람 보내는 식

- 주정차된 차의 이동을 하는 것이 가능한지  =>  이거는 X

- 신고 -> 신고 내용을 API로 제공받음 -> 출동 버튼 -> 사고지까지의 경로 제공 

- 병원 위치 정보에 대한 공공 API를 바탕으로 가까운 병원 매칭

- 알림 상황
 i. 반경 내 모든 사용자에게 무조건적으로 보내는 알람
 ii. 구급차에서 수동으로 보내는 알람
  출동 이후부터는 백그라운드에서 켜져있다는 가정으로 말하는 내용이 수음되고 있는 상태

- 홈 화면에서
 구급차, 병원은 좀 명확한 화면이 있을 것 같은데 
  1            3
 사용자는 지도?? 설정 화면 (알림 on/off)
   2

- 첫 화면은 유저 타입 설정
 구급차  / 병원 / 사용자
 각 기관 코드 입력하면 화면 넘어가는 걸로

- 환자의 수용 여부를 확실히 공유할 수 있는 시스템이 필요함
  신고 내용이 우리 앱에 들어오고 구급차 내에서 음성으로 환자 바이탈이나 사고 경위를 브리핑하면 가까운 병원에 뿌려주고 수용 가능한 병원은 확인 버튼을 누름
